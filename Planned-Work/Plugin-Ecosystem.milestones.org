* Introduction
  :PROPERTIES:
  :initiative_goals: Provide a pluggable contract so consumers can swap among open-source and commercial talking-head/lip-sync providers without changing GuiAssert itself, validated by strong integration tests per plugin (real subprocess invocations or HTTP mock-server tests with exact request-body assertions; live tests gated for credentialed runs).
  :related_specs: [[file:../README.md][GuiAssert README]], [[file:../src/gui_assert/talking_head/core.nim][TalkingHeadProvider contract]]
  :END:

  GuiAssert defines the `TalkingHeadProvider` contract and ships only
  lightweight built-ins (`stock_avatar`). Heavy providers — local ML
  models or commercial APIs — live in sibling repositories named
  `GuiAssert-<Provider>` and are imported by consumers behind a
  `-d:with<Provider>` compile gate.

** Plugin contract
   The contract is `GuiAssert/src/gui_assert/talking_head/core.nim`:

   - `TalkingHeadProvider` value-type with `name`, `isAvailable`, `generate`.
   - `TalkingHeadRegistry` for runtime dispatch by string name.
   - `cacheKeyFor(avatar, narration, providerName, device)` — stable hash.
   - `applyCache(cacheDir, key, outputMp4, generator)` — generic memoisation.
   - `optsFromMetadata(meta, scriptDir)` — translates YAML
     `metadata.talking_head:` blocks into `TalkingHeadOpts`.

   Each plugin exports a `register<Plugin>(reg: TalkingHeadRegistry)` proc
   plus a constructor `<plugin>Provider(): TalkingHeadProvider`.

*** Implementation Notes
    - The runner's =holdRemaining= logic does not currently bound the
      screencast capture to =script.totalDuration=. When a driver step
      runs long (for example, a UI selector that hits the
      Playwright timeout before the target element renders), the
      captured screencast inherits the overrun and the final MP4 ends
      up longer than the script's declared duration. For this
      milestone the produced sample was trimmed back to spec
      via =ffmpeg -t <script.totalDuration> -c copy= (no
      re-encoding, so quality is preserved). *Follow-up*: extend the
      runner to enforce =script.totalDuration= as an upper bound on
      screencast length so post-trimming is not required.
    - =GuiAssert-SadTalker/flake.nix= currently requests =python310=
      from the workspace's transitive =nixpkgs= follow-chain. The
      pinned channel no longer exposes that attribute, so
      =nix develop= in =GuiAssert-SadTalker= fails. The install ran
      against a host Python 3.10 (Homebrew) successfully — every
      other dependency is satisfied by the local venv. *Follow-up*:
      pin the SadTalker flake to a nixpkgs revision that still
      provides =python310=, or migrate the install to a newer
      Python version supported by the patched SadTalker stack.

* Common deliverables per plugin

   Each plugin repo MUST contain:
   - =README.md= (public-safe, no internal references)
   - =LICENSE= (MIT, matches GuiAssert)
   - =.gitignore= (.venv, model-weight extensions, build artefacts)
   - =flake.nix= (dev-shell deps; never bundles heavyweight ML wheels)
   - =Justfile= (minimum =lint= and =test= recipes so workspace hooks pass)
   - =gui_assert_<plugin>.nimble= (requires =gui_assert >= 0.1.0=)
   - =src/gui_assert_<plugin>.nim= (provider contract impl)
   - =tests/t<plugin>.nim= (pure + gated live tests)
   - Documentation of cost / setup steps in README
   - For local providers: =scripts/install.sh= + =scripts/verify-install.sh=
   - Status update in this org file as the milestone completes

* Naming & visibility

   All =GuiAssert-*= plugin repos are *public*. Commit messages must
   stay user-facing — no CodeTracer or codetracer-marketing
   references.

   codetracer-marketing is *private* — its commit messages can be
   richer about internal context.
